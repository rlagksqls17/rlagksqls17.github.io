# github 블로그 만들기 - 1 (소개)   

### 내가 생각하는 블로그란?  

사람들은 누구에게나 '자기 자신의 표현 욕구'가 다양한 방식으로 존재한다. 스스로의 외모를 가꾸거나, 옷을 잘 입거나, 게임에서 실력을 높여 티어를 올린다든가, 자신의 경험과 사진, 느낀 점을 인스타나 페이스북같은 웹에 올리는 것처럼 말이다. 어찌보면 이러한 욕구가 서로 어울리고자 하는 사회성에서 비롯된 당연함일지도 모른다. 나는 블로그를 이용해 자신만의 공간을 꾸미고, 머리를 굴리며 글로 직접 표현하고, 이를 다른 사람들에게 보여주며 소통하는 것도 표현 욕구와 사회성에 포함된다고 생각한다.   

  

블로그는 자기 자신을 세상에게 글로 표현하는 데 아주 유용하다. 웹의 발달로 굳이 다른 사람과 관계를 맺지 않아도 언제든지 검색을 통해 확인이 가능해서 익명의 누군가에게 자기 자신이 어떻게 사는 사람인지, 또는 어떤 것을 주로 공부하는지 설명 가능하기 때문이다. 댓글을 통해 서로 간의 대화도 가능하다. 즉 블로그는 세계의 사람들이 나를 보기 위한, **문서로 된 유리창**이다.  

  

블로그를 운영하기 위한 플랫폼은 네이버, 티스토리, 브런치, 미디엄, 워드프레스, 노션 등 아주 다양하다. 일단 이 글에서는 깃허브와 티스토리를 중점으로 설명하겠지만 다른 좋은 플랫폼들도 많으니 한번 살펴보고 취향껏 선택해보길 바란다. [각 블로그 플랫폼 비교 링크](https://brunch.co.kr/@sijin90/48)

### github.io와 tistory비교  

이제 깃허브와 티스토리를 비교해봄으로써 앞으로 다뤄볼 깃허브 페이지를 소개하도록 하겠다.  

결론부터 말하자면 **티스토리는 사용성 측면에서 매우 우수하고, 깃허브는 자유도가 높다**고 할 수 있겠다. 이제 각 특징별로 비교해보자.  



#### 블로그 생성  

- **github.io** : 블로그 생성에서부터 깃허브는 **진입장벽이 높다**. 후에 포스트하겠지만 먼저 Ruby를 설치하고, Ruby를 이용해서 jekyll 패키지를 컴퓨터에 설치한다. jekyll이 테스트 ip주소에서 잘 운영되는지 검토한 후,  jekyll 테마를 github 레포지토리에서 다운(git clone) 받는다. 이후 본인 id 기반으로 github에 레포지토리를 생성하면, github가 이를 바탕으로 블로그 주소를 생성하게 된다. 이후 github 레포지토리와 자신의 컴퓨터 저장소를 연결(git remote)하고 커밋 후 푸쉬하면 블로그 생성이 완료된다.  
- **tistory** :  티스토리의 경우 먼저 티스토리 계정이나 카카오계정으로 로그인한다. 이후 테마와 주소, 블로그 이름을 쓰면 생성이 완료된다. 사실 여기서 **티스토리의 사용성**이 드러난다.  깃허브와 달리 티스토리는 내 컴퓨터 데이터 기반으로 블로그를 다루는 것이 아닌 티스토리 기업의 데이터 기반으로 블로그를 다루기 때문에, 왠만한 운영들은 티스토리에서 해준다.  

#### **글 작성**  

-  **github.io** : 글은 기본적으로 마크다운(markdown)을 사용한다. 글에 수식을 입력하려면 mathjax라는 자바스크립트 라이브러리를 jeckyll에 적용시켜야 한다. **이미지 업로드가 제일 불편**한데,  [imgur](https://imgur.com/) 에서 내 이미지를 업로드 시켜 받는 특정 링크를 마크다운에 복붙하는게 오류가 제일 안 일어나는 방법이다. (*사실 익숙해지면 제일 쓰기 편한게 마크다운이라 생각한다.*)  

  이후 작성한 글을 .md 파일에 저장한다. git을 공부해본 분들은 아시겠지만 이것을 실제 블로그에 띄우려면 add -> commit -> push 과정을 거쳐야 한다. 이 과정에서 실제로 오류가 많이 일어나기 때문에 **본인은 블로그 작성 시에도 프로그래밍을 하는 것 같다(ㅠㅠ)**. 따라서 한번 글을 작성할 때 실수가 없도록 작성해야 하거나, 아예 branch를 따로 생성해서 잘못된 내용을 사용자가 보는 일이 없도록 해야하는 번거로움이 있다. (아래 처럼 글을 작성하고 올린다.)  

  ![image](https://imgur.com/PIBMKLm.jpg)

  

- **tistory** : 글쓰기 버튼을 클릭하고 글을 작성하면 된다. 티스토리에서는 기본적으로 기본 서체를 제공하고, 마크다운과 html도 추가적으로 지원한다. 이미지 업로드의 경우 간편하게 버튼하나 클릭하고 추가할 이미지를 올리면 된다. (**매우 간편하다**)    

### **추가기능**  

- github  (후에 포스트 예정)
  - 댓글기능은 [disqus](https://disqus.com/) 에서 제공하는 코드를 github.io의 특정 파일에 추가하면 된다.  
  - 조회수의 경우 [Hits](https://hits.seeyoufarm.com/) 에서 자신의 블로그 페이지 링크를 입력한 뒤, 출력되는 링크를 마크다운 페이지에 복사해주면 된다.   
  - 블로그 관리를 위한 통계의 경우 google analytics에서 제공하는 html코드를 블로그에 달아주어야 통계 데이터 확인이 가능하다.  
  - 블로그 수익을 위한 구글 애드센스도 코드로 기입해야 하는데, 일단 본인은 블로그에 광고를 넣을 때 가독성이 너무 떨어져 사용하지 않고, 넣는다 해도 수입이 껌 값 정도로 아주 적기 때문에 선호하지 않아 이에 대한 내용을 잘 모른다. (후에 포스트하도록 하겠다.)  

- tistory  
  - 티스토리에서 댓글기능, 조회수, 블로그 통계와 애드 센스등 추가 기능은 기본적으로 제공되거나 따라하기 쉽게 되어 있다.  github도 당연히 가능하지만 tistory는 자기 취향대로 html코드를 변경해서 기본 스킨을 변경할 수도 있다.

### **정리하며**  

위 글을 보면 알 수 있듯이 깃허브에 비하면 **티스토리는 사용성이 너무 뛰어나다. 그렇다고 네이버 블로그처럼 폐쇄적인 것도 아니고, 디자인도 좀 더 모던하다고 해야할까, 깔끔하다. 또한 블로그 운영하는 입장에서 필요한 부분들은 입맛에 맞게 디자인을 변경할 수 있도록 해놨다. 따라서 좀 더 블로그 작성하는 시간을 효율적으로 사용하고 싶다면 티스토리를 추천한다.** [같은 블로그 다른 느낌](https://mybloglife.tistory.com/2)   



사실 깃허브는 사용성 측면으로 보면 매~~~우 불편하다. 하지만 깃허브는 자유도 측면에서는 최상급으로 높다. **(뭔가 독립하며 자유를 얻은 대신에 빨래나 설거지 등의 일까지 내가 직접 하며 가족의 소중함을 깨닫는 거라고 비유하면 될까)**  

즉 내가 원하는 방식대로 html, css, javascript까지 적용하여 (제한 없이) 웹 디자인이 가능하다.  [이런식으로](https://martianlee.github.io/)   [하지만 첫 상태는 여기서부터다](https://jsanz.github.io/gh-pages-minima-starter/)

또한 내 컴퓨터에 파일을 저장하며 깃허브에 업로드하는 방식이기 때문에 블로그에 저장한 파일이 날라갈까 걱정안해도 된다. 다만 **깃허브의 최대 용량이 1GB**고, 더 늘리려면 매달 돈을 내야하는 것이 치명적으로 와닿는 단점이긴 하다. 



이상으로 티스토리와 깃허브를 소개해보았다. 이 글을 읽고 각자의 취향과 성격에 따라 플랫폼을 잘 선택하는데 도움이 됬으면 좋겠다.
